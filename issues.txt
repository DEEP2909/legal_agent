# Fixed Issues

## #7 — CI release pipeline ✅
Created `.github/workflows/release.yml` that triggers on tags (v*) and push to main.
Builds and pushes API, Worker, and Web images to GHCR with proper tagging.

## #13 — Cursor-based pagination (partial) ✅
Added cursor-based pagination functions to repository.ts:
- `listDocumentsCursor()` - for documents (most critical for 10k+ scale)
- `listResearchHistoryCursor()` - for research history
- Helper functions: `encodeCursor()`, `decodeCursor()`

The existing offset-based functions remain for backward compatibility.
Routes can be gradually migrated to use cursor pagination.

## #16 — Multi-page PDF warning ✅
Added warning in `ocr.ts` when a multi-page scanned PDF is processed via OpenAI Vision:
```
[OCR] Multi-page scanned PDF detected (N pages) but OCR_PROVIDER=openai.
OpenAI Vision only analyzes page 1. For full document OCR, set OCR_PROVIDER=azure_document_intelligence or hybrid.
```

## #19 — /metrics endpoint auth ✅
Added `requirePlatformAdmin` preHandler to the `/metrics` endpoint in `routes.ts`.
Only authenticated platform admins can access Prometheus metrics.

## #20 — services.ts monolith (deferred)
The services.ts split was attempted but deferred due to type incompatibilities between
the service modules and the repository API. The shared types (MfaMethod, LoginResponse, etc.)
and repository method signatures need alignment before this refactor can proceed cleanly.

The services/index.ts now includes a note documenting this:
```typescript
// NOTE: Full domain service split (auth, document, tenant, etc.) is planned but deferred.
// See issues.txt #20 for details. The main services.ts monolith remains the source of truth.
```

This is pure refactoring that doesn't affect functionality - can be done incrementally when time permits.
